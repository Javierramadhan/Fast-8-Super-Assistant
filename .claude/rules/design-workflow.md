# Design Workflow Rules

## Output Standard
Every design output should meet Javier's standard: **clear and detailed**.
- Screens must be annotated or self-explanatory
- Prototypes should be interactive and stakeholder-ready
- Responsive states should be considered by default

## Figma-First
- Primary design tool is Figma
- Figma Desktop MCP is connected — use it for reading/writing designs when possible
- When generating or suggesting UI, default to Figma-compatible thinking (components, auto-layout, tokens)

## Prototype Philosophy
The goal is prototypes stakeholders can *feel*, not just look at.
- Prefer coded prototypes (HTML/CSS/JS) or high-fidelity Figma flows when possible
- Always consider how interactions will feel, not just how they'll look

## Handoff Mindset
Design outputs will go to: PM, Frontend Dev, or UX Writer.
- For devs: include spacing, states, and responsive behavior
- For PMs: include rationale or context in the file/doc
- For UX Writers: flag all copy placeholders clearly
